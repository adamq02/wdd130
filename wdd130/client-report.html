Project Client & Developer Report

Client Report

I focused on the purpose of the site and the target audience. Content was selected to clearly explain the services, while images were chosen to visually support the message and improve engagement without overwhelming the user.

I handled communications by sending my site plan to my lead developer. I tried to make the site plan as clear and concise as possible with wireframes to help with the overall layout of the website and its child pages. This was sent over canvas and he didn't have any questions for me afterwards.

The developers somewhat met my expectations even with the limited information the site plan provided.

I would provide a clearer site plan and also give the developers more creative freedom. I would let them decide how to style the layout while I would provide the color scheme.

Site link: https://hyrummaughan.github.io/wdd130-Querubin/

Lead Developer Report

I worked on the main homepage which included the header and footer which would be copied by the junior developers for their child pages. I delegated one child page each to my teammates ahead of time so that they would have adequate time to work on their pages. I also instructed that we each use our own separate styling pages with only the header and footer code the same so that we would not overlap any code and maybe mess up each other's pages.

Weekly updates and demos were used to keep the client informed with feedback directly from the client helping in the development process to accurately deliver the website he wanted. If not in person, we communicated through email as well where I sent him the URL so he could see any updates made online.

The site was deployed using GitHub Pages and the live link was shared with the client after testing. I also made use of the Github Desktop application to simplify the process of pushing/pulling origins which made the collaboration alot easier.

Site link: https://adamq02.github.io/wdd130-cook/index.html

Junior Developer Report

Communication with my lead developer was mostly done through canvas messaging. Instructions were clear and concise so it made focusing on my work easier.

The tasks were clear, work on your respective child pages using the same footer and header that your lead developer created and also based off the site plan wireframes.

I feel like I was able to fulfill my role as both Lead developer and Junior developer in each team. I created each child page and styled them according to the site plan.

The only issues I had with GitHub was the initial set up with the repositories functioning.

General Experience and Reflections

Getting in contact with lead and junior developers was probably the most challenging aspect as the groups were mostly randomised so communication could only happen through emails or Canvas.

I enjoyed seeing how Github allowed me to see live updates of when people were making changes so it truly felt like aa collaborative effort.

I would plan more time for testing and feedback before starting work on each website.

This project helped familiarise Github for me which is a very useful tool for the real world as group development work is more clear to me.
